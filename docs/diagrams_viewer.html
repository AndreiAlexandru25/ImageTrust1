What is ImageTrust / Cyber Scout?

ImageTrust (commercially named Cyber Scout) is a forensic application for detecting AI-generated images and identifying digital manipulations.

Problems Solved:

  • AI Detection: Identifies images generated by DALL-E, Midjourney, Stable Diffusion
  • Manipulation Detection: ELA, noise analysis, JPEG artifacts, resampling
  • Metadata Analysis: EXIF, XMP, software traces (Photoshop, etc.)
  • Source Attribution: Hints for screenshot, WhatsApp, Instagram (probabilistic)
  • Explainability: Grad-CAM heatmaps, per-plugin evidence

Technology Stack:

Python 3.10+ PyTorch HuggingFace Transformers FastAPI Streamlit pywebview PyInstaller OpenCV Pillow

Deliverables:

  • Desktop App: CyberScout.exe - portable Windows application
  • Web UI: Streamlit interface in the browser
  • CLI: Command-line tool for automation
  • REST API: FastAPI for external integrations
  • Reports: Export to JSON, Markdown, PDF

The plugin-based separation enables:

  • Independent testing of each module
  • Flexibility in choosing analyses (--pixel, --metadata, --ai, etc.)

Detailed Explanation

What does this diagram show?

The decision logic for the final verdict. The system does NOT perform simple binary classification (real/fake), but instead computes probabilities for multiple labels and includes safety mechanisms to avoid false verdicts.

Steps in Detail:

  1. Aggregate Scores: Collects results from all plugins
  2. Compute Label Probabilities: Calculates the probability for each label:
    • P(AI_GENERATED) - based on HF model scores + frequency signals
    • P(EDITED) - based on ELA, software traces, noise inconsistency
    • P(CAMERA_ORIGINAL) - based on valid EXIF + absence of anomalies
    • P(SCREENSHOT) - based on resolution, aspect ratio
    • P(SOCIAL_MEDIA) - based on platform compression signatures
  3. Threshold Check (0.4): If NO probability exceeds 0.4 -> UNKNOWN
  4. Contradiction Check: Verifies inconsistencies:
    • "Has camera EXIF" + "High AI score" = Contradiction
    • "Photoshop in metadata" + "No pixel evidence" = Suspicion
  5. Select Verdict: Chooses the label with the highest probability
  6. Calibration: Applies Temperature Scaling for more reliable probabilities
Why UNKNOWN/INCONCLUSIVE?

The system is designed to avoid false positives. It is better to say "I don't know" than to falsely accuse an authentic image of being AI-generated.

Conditions for UNKNOWN:

  • No probability > 0.4 (insufficient evidence)
  • Severe contradictions between signals
  • Critical plugins could not run (e.g., not a JPEG)

Output Examples:

Scenario Probabilities Verdict
Clear DALL-E image AI: 0.91, Edit: 0.15, Camera: 0.05 AI_GENERATED (HIGH)
Camera photo with EXIF AI: 0.08, Edit: 0.12, Camera: 0.85 CAMERA_ORIGINAL (HIGH)
WhatsApp screenshot AI: 0.20, Screenshot: 0.72, Social: 0.65 SCREENSHOT (MEDIUM)
Ambiguous image AI: 0.35, Edit: 0.30, Camera: 0.25 UNKNOWN (LOW)

What does this diagram show?

How the distributable desktop application (CyberScout.exe) works. It is a native wrapper over the Streamlit web interface.

Components:

Component Technology Role
CyberScout.exe PyInstaller bundle Standalone Windows executable, does not require Python installed
desktop_launcher.py Python Entry point - starts the server and the window
Streamlit Server Streamlit Local web server (localhost:8501) with the application UI
pywebview pywebview Creates a native OS window that displays web content
WebView2 Microsoft Edge Rendering engine (pre-installed on Windows 10/11)
HF Models HuggingFace cache Models are downloaded on first run and cached locally

Startup Flow:

  1. User double-clicks CyberScout.exe
  2. desktop_launcher.py executes
  3. Starts Streamlit server on localhost:8501 (separate thread)
  4. Creates a pywebview window that loads localhost:8501
  5. User interacts with the UI in the native window
  6. During analysis, Streamlit calls the Forensics Engine
  7. Engine uses HF models from the local cache
Advantages:
  • Offline: Works without internet (after initial model download)
  • Portable: No installation required, just run the .exe
  • Native: Looks like a normal Windows application, not a browser
  • Private: All analyses remain on the user's PC
WebView2 Dependency:

WebView2 Runtime is required and comes pre-installed on recent Windows 10/11. If missing, the application displays a dialog with a download link.

Fallback: If WebView2 cannot be installed, the UI opens in the default browser.

Evaluation Metrics:

Additional Evaluations (thesis requirement):

  • Cross-Generator: Train on Midjourney -> Test on DALL-E/SD (generalization)
  • Degradation: Performance under JPEG compression, resize, blur, noise
  • Ablation Study: Contribution of each component
Why compare with baselines?
  • Validates that our system adds value over simple methods
  • Identifies where each approach excels

7. Technical Summary

ImageTrust / Cyber Scout - Summary

What the system does:

  1. Receives an image (JPEG, PNG, WebP)
  2. Runs 12+ forensic analyses in parallel (pixel, metadata, source, AI)
  3. Aggregates results into a calibrated multi-label verdict
  4. Generates a report with evidence and limitations

Key Architecture:

  • Plugin-based: Each analysis is an independent plugin
  • Multi-label: Not just "real/fake" but probabilities for 7+ categories
  • Calibrated: Probabilities adjusted for reliability (Temperature Scaling)
  • Conservative: Prefers UNKNOWN over false positives

Software Components:

Layer Components
UI CLI (Click), Web (Streamlit), Desktop (pywebview), API (FastAPI)
Core ForensicsEngine, FusionLayer, Calibration
Plugins ELA, Noise, JPEG, Metadata, Screenshot, Platform, AI (4 HF models)
Output JSON, Markdown, PDF reports + artifact heatmaps

Thesis Requirements Fulfilled:

  • Comparison with 3 baselines (Classical, CNN, ViT)
  • Calibration (Temperature Scaling, ECE metrics)
  • UNCERTAIN region for insufficient evidence
  • Ablation study (contribution of each component)
  • Documented threat model
  • Reproducibility (fixed seeds, splits)
