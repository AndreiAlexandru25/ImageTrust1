🎯 Ce este ImageTrust / Cyber Scout?

ImageTrust (denumit comercial Cyber Scout) este o aplicație forensică pentru detectarea imaginilor generate de inteligență artificială și identificarea manipulărilor digitale.

Probleme rezolvate:

  • Detectare AI: Identifică imagini generate de DALL-E, Midjourney, Stable Diffusion
  • Detectare Manipulări: ELA, analiză noise, artefacte JPEG, resampling
  • Analiză Metadata: EXIF, XMP, trasee software (Photoshop, etc.)
  • Atribuire Sursă: Hint-uri pentru screenshot, WhatsApp, Instagram (probabilistic)
  • Explicabilitate: Grad-CAM heatmaps, evidențe per-plugin

Stack Tehnologic:

Python 3.10+ PyTorch HuggingFace Transformers FastAPI Streamlit pywebview PyInstaller OpenCV Pillow

Livrabile:

  • Desktop App: CyberScout.exe - aplicație Windows portabilă
  • Web UI: Interfață Streamlit în browser
  • CLI: Tool linie de comandă pentru automatizare
  • REST API: FastAPI pentru integrări externe
  • Rapoarte: Export JSON, Markdown, PDF

Arhitectura completă a sistemului, de la input (imagine) până la output (rapoarte). Sistemul este modular, cu separare clară între interfețe, motor de analiză și output.

Componente principale:

Componentă Tehnologie Rol

Forensics Engine - Inima sistemului:

  1. Plugin Orchestrator: Încarcă și coordonează toate plugin-urile de analiză
  2. 4 Pack-uri de Plugin-uri:
    • Pixel Pack: ELA, Noise Analysis, JPEG Artifacts, Resampling, Edge Halo
    • Metadata Pack: EXIF parsing, XMP, Software traces, Thumbnail mismatch
    • Source Pack: Screenshot detection, Platform hints, Compression history
    • AI Pack: 4 modele HuggingFace + analiză frecvență/textură
  3. Fusion Layer: Agregă scorurile și decide verdictul final
  4. Calibration: Ajustează probabilitățile pentru a fi mai fiabile (Temperature Scaling)
💡 De ce această arhitectură?
Separarea în plugin-uri permite:
  • Adăugarea ușoară de noi detectori
  • Testarea independentă a fiecărui modul
  • Flexibilitate în alegerea analizelor (--pixel, --metadata, --ai, etc.)

Ce arată această diagramă?

Structura de clase pentru sistemul de plugin-uri. Fiecare plugin implementează interfața ForensicsPlugin și returnează un rezultat standardizat ForensicsResult.

Clasa abstractă ForensicsPlugin:

ForensicsResult - Rezultat standardizat:

  • explanation: str - Explicație în limbaj natural
  • limitations: List[str] - Limitări cunoscute ale analizei
  • artifacts: List[Artifact] - Heatmap-uri, grafice generate

📖 Explicație Detaliată

Ce arată această diagramă?

Logica de decizie pentru verdictul final. Sistemul NU face clasificare binară simplă (real/fake), ci calculează probabilități pentru multiple label-uri și include mecanisme de siguranță pentru a evita verdicte false.

Pași în detaliu:

  1. Aggregate Scores: Colectează rezultatele de la toate plugin-urile
  2. Compute Label Probabilities: Calculează probabilitatea pentru fiecare etichetă:
    • P(AI_GENERATED) - bazat pe scorurile modelelor HF + semnale frecvență
    • P(EDITED) - bazat pe ELA, software traces, noise inconsistency
    • P(CAMERA_ORIGINAL) - bazat pe EXIF valid + lipsa anomaliilor
    • P(SCREENSHOT) - bazat pe rezoluție, aspect ratio
    • P(SOCIAL_MEDIA) - bazat pe semnături de compresie platforme
  3. Threshold Check (0.4): Dacă NICIO probabilitate nu depășește 0.4 → UNKNOWN
  4. Contradiction Check: Verifică inconsistențe:
    • "Are EXIF de cameră" + "Scor AI mare" = Contradicție
    • "Photoshop în metadata" + "Fără evidențe pixel" = Suspiciune
  5. Select Verdict: Alege eticheta cu probabilitatea maximă
  6. Calibration: Aplică Temperature Scaling pentru probabilități mai fiabile
⚠️ De ce UNKNOWN/INCONCLUSIVE?

Sistemul este conceput să evite false positive-uri. Este mai bine să spunem "Nu știu" decât să acuzăm fals o imagine autentică de a fi generată de AI.

Condiții pentru UNKNOWN:

  • Nicio probabilitate > 0.4 (evidențe insuficiente)
  • Contradicții severe între semnale
  • Plugin-uri critice nu au putut rula (ex: nu e JPEG)

Exemple de output:

Scenariu Probabilități Verdict
Imagine DALL-E clară AI: 0.91, Edit: 0.15, Camera: 0.05 AI_GENERATED (HIGH)
Foto cameră cu EXIF AI: 0.08, Edit: 0.12, Camera: 0.85 CAMERA_ORIGINAL (HIGH)
Screenshot WhatsApp AI: 0.20, Screenshot: 0.72, Social: 0.65 SCREENSHOT (MEDIUM)
Imagine ambiguă AI: 0.35, Edit: 0.30, Camera: 0.25 UNKNOWN (LOW)

Ce arată această diagramă?

Cum funcționează aplicația desktop distribuibilă (CyberScout.exe). Este un wrapper nativ peste interfața web Streamlit.

Componente:

Componentă Tehnologie Rol
CyberScout.exe PyInstaller bundle Executabil Windows standalone, nu necesită Python instalat
desktop_launcher.py Python Entry point - pornește server-ul și fereastra
Streamlit Server Streamlit Server web local (localhost:8501) cu UI-ul aplicației
pywebview pywebview Creează fereastră nativă OS care afișează conținut web
WebView2 Microsoft Edge Engine de rendering (pre-instalat pe Windows 10/11)
HF Models HuggingFace cache Modelele sunt descărcate la prima rulare și cached local

Flow la pornire:

  1. User dă dublu-click pe CyberScout.exe
  2. desktop_launcher.py se execută
  3. Pornește Streamlit server pe localhost:8501 (thread separat)
  4. Creează fereastră pywebview care încarcă localhost:8501
  5. User interacționează cu UI-ul în fereastră nativă
  6. La analiză, Streamlit apelează Forensics Engine
  7. Engine folosește modelele HF din cache local
✅ Avantaje:
  • Offline: Funcționează fără internet (după download inițial modele)
  • Portabil: Nu necesită instalare, doar rulezi .exe
  • Nativ: Arată ca aplicație Windows normală, nu browser
  • Privat: Toate analizele rămân pe PC-ul utilizatorului
⚠️ Dependența WebView2:

WebView2 Runtime este necesar și vine pre-instalat pe Windows 10/11 recent. Dacă lipsește, aplicația afișează dialog cu link de download.

Fallback: Dacă WebView2 nu se poate instala, UI-ul se deschide în browser-ul default.

📖 Explicație Detaliată

Ce arată această diagramă?

Protocolul de evaluare pentru teză. Cerință obligatorie: Compararea sistemului nostru cu minimum 3 baseline-uri, folosind același dataset, aceleași metrici, aceleași split-uri.

Baseline-urile implementate:

Baseline Tehnică Caracteristici Limitări
B1: Classical LogReg / XGBoost Features: DCT stats, noise profile, edge histogram, color stats Nu capturează pattern-uri complexe
B2: CNN ResNet-50 Pretrained ImageNet, fine-tuned pe real/AI Single model, prone to overfit
B3: ViT Vision Transformer CLIP-ViT-L, attention-based Computationally expensive
ImageTrust (Ours) Ensemble + Fusion 4 HF models + pixel + metadata + source + calibration Complex, dar robust

Metrici de evaluare:

  • Accuracy: Procentul de clasificări corecte
  • F1 Score: Media armonică între precision și recall
  • AUC-ROC: Aria sub curba ROC (discriminare)
  • ECE (Expected Calibration Error): Cât de bine corespund probabilitățile cu realitatea

Evaluări adiționale (cerință teză):

  • Cross-Generator: Train pe Midjourney → Test pe DALL-E/SD (generalizare)
  • Degradation: Performanță sub JPEG compression, resize, blur, noise
  • Ablation Study: Contribuția fiecărei componente
💡 De ce comparăm cu baseline-uri?
  • Validează că sistemul nostru aduce valoare peste metode simple
  • Identifică unde fiecare abordare excelează
  • Oferă context pentru rezultatele obținute
  • Cerință academică obligatorie pentru disertație

7. Sumar Tehnic

🎯 ImageTrust / Cyber Scout - Rezumat

Ce face sistemul:

  1. Primește o imagine (JPEG, PNG, WebP)
  2. Rulează 12+ analize forensice în paralel (pixel, metadata, source, AI)
  3. Agregă rezultatele într-un verdict multi-label calibrat
  4. Generează raport cu evidențe și limitări

Arhitectură cheie:

  • Plugin-based: Fiecare analiză este un plugin independent
  • Multi-label: Nu doar "real/fake" ci probabilități pentru 7+ categorii
  • Calibrated: Probabilități ajustate pentru fiabilitate (Temperature Scaling)
  • Conservative: Preferă UNKNOWN decât false positive

Componente software:

Nivel Componente
UI CLI (Click), Web (Streamlit), Desktop (pywebview), API (FastAPI)
Core ForensicsEngine, FusionLayer, Calibration
Plugins ELA, Noise, JPEG, Metadata, Screenshot, Platform, AI (4 HF models)
Output JSON, Markdown, PDF reports + artifact heatmaps

Cerințe teză îndeplinite:

  • ✅ Comparație cu 3 baseline-uri (Classical, CNN, ViT)
  • ✅ Calibrare (Temperature Scaling, ECE metrics)
  • ✅ Regiune UNCERTAIN pentru evidențe insuficiente
  • ✅ Ablation study (contribuția fiecărei componente)
  • ✅ Threat model documentat
  • ✅ Reproducibilitate (fixed seeds, splits)
